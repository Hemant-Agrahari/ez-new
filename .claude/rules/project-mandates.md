# EZ-Plumbing Project Mandates & Memory

This document is the **Single Source of Truth** for project standards and must be read before responding to any prompt.

## 🚫 Banned Libraries (removed April 2026)

NEVER re-add or use these:

- **jQuery**: Use Vanilla DOM APIs.
- **Slick Carousel**: Use `VCarousel` from `carousel-vanilla.js`.
- **Bootstrap JS**: Use `VModal` from `modal-vanilla.js`.
- **AOS**: Use `aos-init.js` (IntersectionObserver + CSS).
- **Font Awesome**: Use inline SVGs.

## ⚡ Core Web Vitals (CWV) Standards

- **LCP**: Hero images must NOT be lazy-loaded. Use explicit `width`/`height` and `fetchpriority="high"`.
- **CLS**: Zero layout shifts. Critical CSS **inlined** in `<head>` (<14KB gzipped). Per-page purged CSS **deferred**.
- **INP**: Passive event listeners. Heavy work in `requestIdleCallback`.

## 🎨 CSS Strategy

- **File Hierarchy**: `shared/css/vendor/` (source) → `shared/css/global.css` (overrides) → `shared/css/pages/{slug}.css` (per-page).
- **Purging**: All vendor CSS (including Bootstrap) is purged per-page.
- **Critical CSS**: Extracted and inlined by `tools/extract-critical-css-page.js`.

## 📜 JavaScript Strategy

- **Defer everything**: Every `<script src>` must use `defer`.
- **Modular**: One feature per module. Avoid monolithic `animations.js`.
- **No frameworks**: Stick to vanilla JS (<5KB libraries only).

## 🔍 SEO & Metadata

- **Canonical Order**: Follow `docs/conventions/conventions/head-order.md`.
- **Tags**: Single `<h1>`, `<title>`, `<meta description>`, `<link canonical>`, and OG tags for social.

## 🔄 Next.js Conversion Rules

- **Asset Paths**: Convert `/_next/image` proxy URLs to local `/assets/` paths.
- **URL Cleanup**: Strip Next.js URL params (e.g., `&w=48&q=75`).
- **Build**: Run `node build.js` to propagate changes to `dist/`.
